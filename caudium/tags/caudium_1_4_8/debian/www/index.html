Unless you changed its configuration, your new server is configured as follows:

  • Configuration files can be found in /etc/caudium
  • The default virtual host root, which is the directory under which all your HTML files should exist, is set to /var/www
  • CGI scripts are looked for in /usr/lib/cgi-bin, which is where Debian packages will place their scripts. The URL configured for this directory is /cgi-bin/
  • Log files are placed in /var/log/caudium/ and its subdirectories, and will be rotated monthly. The frequency of rotation can be easily changed by editing /etc/logrotate.d/caudium
  • The default directory index is index.html, meaning that requests for a directory /foo/bar/ will give the contents of the file /var/www/foo/bar/index.html if it exists (assuming that /var/www is your virtual host root).
  • User directories are enabled, and user documents will be looked for in the public_html directory of the users' homes.

All the standard Caudium modules are found in the Caudium packages as well as a few pre-defined virtual host configurations. Adding new modules is as simple as opening the Caudium Configuration Interface in your browser by connecting to the port you have configured for it during the installation process.

More documentation on Caudium can be found on:
